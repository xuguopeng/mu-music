## 待处理

### 1. 刚开始播放时喜欢按钮不可用

- 模块：Migi 客户端 / 播放器
- 现象：歌曲刚开始播放时点击“喜欢”不起作用，需要听上一会儿后才可以点击喜欢。
- 关联现象：从歌词面板切换到聊天面板时也存在交互状态不顺的问题。
- 初步判断：可能和当前播放歌曲状态、喜欢接口调用时机、播放器初始化状态或面板切换后的 UI 状态同步有关。
- 优先级：中高
- 状态：待周更时排查和修复

### 2. 聊天滚动条停在最上方

- 模块：Migi 客户端 / 聊天面板
- 现象：聊天滚动条一直在最上面，需要手动滑下来才能看到最新内容。
- 期望：进入聊天、发送消息、收到 Migi 回复、从歌词切换回聊天时，默认自动滚动到最新消息；用户手动向上查看历史时不要强制抢滚动。
- 初步判断：可能和 `ScrollController` 初始化、面板切换后布局高度变化、消息追加后的延迟滚动时机有关。
- 优先级：中高
- 状态：待周更时排查和修复

### 3. 所有歌曲的专辑名称显示不出来

- 模块：Migi 客户端 / 音乐元数据展示
- 现象：所有歌曲的专辑名称都没有正常显示。
- 期望：播放器区域、播放列表、歌词/聊天切换区域里都能显示真实专辑名称；没有专辑时再显示兜底文案。
- 初步判断：可能和 NAS `albumTitle/album/name/al.name` 字段映射、客户端 `normalizeTrack`、播放状态 `currentTrack` 同步或 UI 读取字段不一致有关。
- 优先级：中高
- 状态：待周更时排查和修复

### 4. 本地有歌但聊天点播判断为没有

- 模块：NAS 服务端 / Migi 聊天点播检索
- 现象：本地曲库里已有歌曲，但聊天输入“播放 炉火”“播放 主角”时，Migi 回复当前 `recentTracks` 和 180 天画像里没有确认命中，先用相似歌曲接住。
- 复现：客户端聊天输入“播放 炉火”“播放 主角”；截图显示当前播放器能播放“主角 - 王菲”，说明至少“主角”实际存在。
- 期望：点播命令应优先查完整本地曲库，包括标题、文件名、路径、别名/繁简/空格符号等；本地有歌时直接播放，不应被最近记录或画像样本限制。
- 搜索优化要求：聊天点播后，后台先做全库强匹配和模糊匹配；如果确实没有命中，就查出最相关的前 10 首歌曲作为候选推荐给用户，而不是直接说没有。
- 初步判断：可能是 `/v1/dj/chat` 点播意图解析只看 `recentTracks` / 画像样本，或服务端强匹配没有调用完整曲库搜索；也可能是标题清洗、精确匹配和模糊匹配阈值的问题。周更时需要把“播放动作”和“候选推荐”都做成结构化 action，客户端可以直接展示/点击播放候选歌曲。
- 优先级：高
- 状态：待周更时排查和修复
